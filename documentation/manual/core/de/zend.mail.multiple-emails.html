Versand von mehreren E-Mails über eine SMTP Verbindung

Standardmäßig erstellt ein einzelner SMTP Transport eine einzelne Verbindung und verwendet Sie wieder wärend der Laufzeit der Skriptausführung. Es können mehrere E-Mails über diese SMTP Verbindung gesendet werden. Ein RSET Kommando wird vor jeder Ausführung angewendet um sicherzustellen das der korrekte SMTP Handschlag folgt.

Optional kann auch eine standardmäßige From Emailadresse und Name definiert werden sowie ein standardmäßiger reply-to Header. Das kann durch die statischen Methoden setDefaultFrom() und setDefaultReplyTo() getan werden. Diese Standardwerte werden verwendet wenn man keine From oder Reply-to Adresse oder -Name angibt bis die Standardwerte zurückgesetzt werden (gelöscht). Das Zurücksetzen der Standardwerte kann durch die Verwendung von clearDefaultFrom() und clearDefaultReplyTo durchgeführt werden.

Example #1 Versand von mehreren E-Mails über eine SMTP Verbindung

  1. // Transport erstellen
  2. $config = array('name' => 'sender.example.com');
  3. $transport = new Zend_Mail_Transport_Smtp('mail.example.com', $config);
  4.  
  5. // Setzt From & Reply-To Adressen
  6. // und Namen für alle Emails die zu versenden sind
  7. Zend_Mail::setDefaultFrom('sender@example.com', 'John Doe');
  8. Zend_Mail::setDefaultReplyTo('replyto@example.com','Jane Doe');
  9.  
  10. // Durch die Nachrichten gehen
  11. for ($i = 0; $i < 5; $i++) {
  12.     $mail = new Zend_Mail();
  13.     $mail->addTo('studio@example.com', 'Test');
  14.     $mail->setFrom('studio@example.com', 'Test');
  15.     $mail->setSubject(
  16.         'Demonstration - mit einer SMTP Verbindung mehrfache E-Mails senden'
  17.     );
  18.     $mail->setBodyText('...Hier die Nachricht...');
  19.     $mail->send($transport);
  20. }
  21.  
  22. // Setzt die Standardwerte zurück
  23. Zend_Mail::clearDefaultFrom();
  24. Zend_Mail::clearDefaultReplyTo();

Wenn man eine separate Verbindung für jeden Mailtransport haben will, muß der Transport vor und nach jedem Aufruf der send() Methode erstellt und gelöscht werden. Oder alternativ kann die Verbindung zwischen jedem Transport, durch Zugriff auf das Protokoll Objekt des Transportes, manipuliert werden.

Example #2 Die Transportverbindung manuell kontrollieren

  1. // Transport erstellen
  2. $transport = new Zend_Mail_Transport_Smtp();
  3.  
  4. $protocol = new Zend_Mail_Protocol_Smtp('mail.example.com');
  5. $protocol->connect();
  6. $protocol->helo('mail.example.com');
  7.  
  8. $transport->setConnection($protocol);
  9.  
  10. // Durch die Nachrichten gehen
  11. for ($i = 0; $i < 5; $i++) {
  12.     $mail = new Zend_Mail();
  13.     $mail->addTo('studio@example.com', 'Test');
  14.     $mail->setFrom('studio@example.com', 'Test');
  15.     $mail->setSubject(
  16.         'Demonstration - mit einer SMTP Verbindung mehrfache E-Mails senden'
  17.     );
  18.     $mail->setBodyText('...Hier die Nachricht...');
  19.  
  20.     // Die Verbindung manuell kontrollieren
  21.     $protocol->rset();
  22.     $mail->send($transport);
  23. }
  24.  
  25. $protocol->quit();
  26. $protocol->disconnect();
