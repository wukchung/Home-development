Zeichensätze

Zend_Mail prüft nicht auf die richtigen Zeichensätze für die Mailbestandteile. Wenn Zend_Mail instanziiert wird, kann ein Zeichensatz für die Mail selbst angegeben werden. Der Standardwert ist iso-8859-1. Die Anwendung muss sicherstellen, dass die Inhalte aller Bestandteile des Mailobjektes im richtigen Zeichensatz codiert sind. Wenn ein neuer Mailbestandteil erstellt wird, kann für jeden Bestandteil ein anderer Zeichensatz angegeben werden.

Note: Nur im Textformat
 Zeichensätze sind nur auf Mailbestandteile im Textformat anwendbar.

Example #1 Verwendung in CJK Sprachen

Das folgende Beispiel zeigt wie Zend_Mail im japanischen verwendet werden kann. Das ist eine der CJK (oder auch CJKV) Sprachen. Wenn man Chinesisch verwendet, kann man auch HZ-GB-2312 statt ISO-2022-JP verwenden.

  1. // Wir nehmen an das die Zeichenkodierung der Strings in PHP Skripten UTF-8 ist
  2. function myConvert($string) {
  3.     return mb_convert_encoding($string, 'ISO-2022-JP', 'UTF-8');
  4. }
  5.  
  6. $mail = new Zend_Mail('ISO-2022-JP');
  7. // In diesem Fall kann ENCODING_7BIT verwendet werden,
  8. // weil ISO-2022-JP MSB nicht verwendet
  9. $mail->setBodyText(myConvert('This is the text of the mail.'),
  10.                              null,
  11.                              Zend_Mime::ENCODING_7BIT);
  12. $mail->setHeaderEncoding(Zend_Mime::ENCODING_BASE64);
  13. $mail->setFrom('somebody@example.com', myConvert('Some Sender'));
  14. $mail->addTo('somebody_else@example.com', myConvert('Some Recipient'));
  15. $mail->setSubject(myConvert('TestSubject'));
  16. $mail->send();
